Cliver Excel 7.*

Overview

CliverExcel is a .NET lib for creating and editing excel files.
It is based on the NPOI lib and basically is a collection of extension methods to the NPOI objects. Its goal is providing an API more advanced and handy than NPOI's.

Compatibility
CliverExcel is compatible with .NET Standard 2.0.

Conception

Most methods in CliverExcel are presented as extension methods for the NPOI types: IWorkbook, ISheet, IRow and ICell.

(!)All the CliverExcel extension methods start with '_' to be easily distinguished from the NPOI natives.

Besides the extension methods, there are classes that provide routines native to CliverExcel. The main such class is Cliver.Excel.

(!)In CliverExcel, cell, row and sheet indexes are always 1-based, while in the NPOI API they are mostly 0-based. As both API's coexist, be careful of the arguments and output of the methods you are using.

In the CliverExcel, row and column indexes are always represented by 'y' and 'x'.

Cliver.Excel

Any work with CliverExcel must begin with creation of a Cliver.Excel.

Creating a Cliver.Excel means creating both Cliver.Excel and IWorkbook objects at once which remain coupled during their lifetime. These objects reciprocally allow accessing each other by the respective method/property.

Cliver.Excel provides properties and methods that could not be implemented by the mechanism of C# extension methods and thus can be viewed as a sort of 'extension property' of IWorkbook if such a mechanism existed in C#.

(!)CliverExcel extension methods can access the Cliver.Excel object via IWorkbook, ISheet, IRow or ICell passed in them; hence, such objects must be originated through Cliver.Excel, not by the NPOI API directly.

Excel.Range

The class Excel.Range provides routines for ranges. It is native to CliverExcel and has no underlying NPOI type.

Excel.Column

The class Excel.Column looks to provide routines for sheet columns analogous to those provided for IRow. It has no underlying NPOI type.

Excel.StyleCache

StyleCache helps when you need to alter certain style parameters for some cells, e.g. set a new color or font, but you do not know in advance which styles you will alter and hence, which new styles must be created.

StyleCache takes care of registering and caching all the styles needed in the workbook during editing.

Profits by StyleCache:
- guaranteeing no style duplication;
- increasing performance;

A Cliver.Excel always has a StyleCache object which is used by default when operating within the same workbook. One StyleCache is enough for one IWorkbook.
